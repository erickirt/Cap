Cap feedback stays attached to the recording and playback moment it refers to. A signed-in viewer with access can post comments, reply to a thread, or add an emoji reaction when those features are enabled.

## Leave a timestamped comment

1. Open the Cap and sign in if needed.
2. Play or scrub to the relevant moment.
3. Open the comment composer from the viewer toolbar.
4. Enter the feedback and submit it.

The comment stores the current playback timestamp. Selecting its timestamp returns the player to that point.

## Reply and react

Replies keep a discussion under its original comment. Emoji reactions provide a lightweight response tied to the Cap and timestamp.

Organization or effective viewer settings can disable comments and reactions. When disabled, the corresponding controls are not available to viewers.

## Notifications

New feedback can create in-app and email notifications for the recording owner or relevant participants. Delivery depends on the event and each account's notification preferences.

## Delete feedback

The share-page UI lets a comment author delete their own comment. Do not assume that owning a recording automatically exposes a delete control for every other person's comment in that UI.

Agents and other authorized API clients must follow the permissions returned by Cap and ask for confirmation before deleting comments or reactions.

## Work with an agent

An agent can read a Cap's context, including available comments and reactions, then draft a response at a specific timestamp. Ask it to show the exact text and timestamp before posting.

See [Agent Workflows](/docs/agents/workflows#draft-and-post-a-comment) for a safe prompt and command pattern.
